interactieve video

Inleiding

Voor ons mini-museum willen we ervoor zorgen dat er video’s te zien zijn op het museum. Om de bezoeker beter te betrekken bij het museum hebben we besloten om de video-speler interactief te maken.

Hieronder is het proces te lezen over hoe ik deze applicatie heb opgebouwd, tegen welke problemen ik aan ben gelopen, wat het uiteindelijke resultaat is geworden en wat de verbeterpunten zijn voor verdere ontwikkeling.

User story

Dit is de user story waar ik tijdens sprint 3 aan gewerkt heb: Als student wil ik dat de video mij relevante opties kan laten zien in verband met het onderwerp van de video. In de afbeelding hieronder staan de taken die behoren tot de user story.

Het volledige overzicht aan taken voor sprint 3 is hier terug te vinden op het Trello bord.

Proces

Ik ben begonnen met het maken van een lijstje waarin ik alle onderdelen heb benoemd die deze applicatie moest kunnen. Zo moest er onder andere een auto-play functie zijn, er moesten verschillende opties beschikbaar worden aan het einde van iedere video en moest de video standaard op full-screen afgespeeld worden.

Toen ik de functionele eisen voor ogen had ben ik begonnen met het opzetten van een Git repository en heb ik een development branch aangemaakt. Toen heb ik via VS Code mijn Git gekoppeld zodat ik vanuit VS Code mijn repository kon aansturen. Tijdens het programmeren heb ik eerst de structuur van de pagina opgebouwd en ben ik daarna begonnen met het implementeren van de functies. Deze waren over het algemeen geen probleem.

Nadat ik deze functionaliteiten had geïmplementeerd, heb ik feedback gevraagd. Bij deze feedback zaten een hele boel tips over hoe ik mijn code kon opschonen, fouten eruit kon halen, wat beter was om te gebruiken, hoe ik het beste mijn vervolgstappen kon zetten en hoe ik de applicatie verder kon uitbreiden.

Deze feedback heb ik dan ook zoveel mogelijk proberen te verwerken. Waar ik het meest moeite mee had was toen ik mijn code modulair moest maken. Hiervoor heb ik gelukkig dan ook hulp kunnen zoeken bij mijn groepsgenoten. Ze hebben mij goed geholpen met het begrijpen van het opstellen van modulaire code. Hierdoor heb ik een module kunnen maken van mijn code zodat deze ook gebruikt kan worden in andere externe projecten.

Hier is een link naar de Git repository van het project.

De vervolgstappen van deze applicatie zijn om deze te koppelen met het Dashboard wat Daan heeft gebouwd. Hierdoor wordt voor de gebruiker mogelijk om zelf de tekst en linkjes naar andere video’s te bepalen. Dit geeft de gebruiker meer vrijheid.

Reflectie

Aangezien mijn javascript ervaring nog altijd verbeterd kan worden was dit een prima opdracht om mijn ervaring te laten groeien. Dit was namelijk een perfecte opdracht om de kennis die ik al had te combineren met kennis die ik nog moest opdoen. Ik ben ook blij met het uiteindelijke resultaat dat ik opgeleverd heb. Zo heb ik geleerd hoe ik een module kan schrijven en hierdoor ook mijn code gesplitst kan houden. Daarnaast heb ik geleerd hoe ik mijn code beter kan schrijven zodat deze korter en beter leesbaar is.
