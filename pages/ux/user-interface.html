User Interface

Inleiding

Vanuit het vak van UX was er een individuele opdracht waarbij een prototype vormgegeven moest worden tot een interactieve grafische userinterface. Ik heb hierbij de keuze gemaakt om een prototype te maken voor het aanmelden van een nieuwsbrief via een website. Dit is een onderdeel dat van pas komt bij het concept dat we aan het realiseren zijn.

Het proces

Ik ben begonnen met het kijken naar de website van Brainport Eindhoven zelf. Hier heb ik de stijl bekeken en hoe de website is opgebouwd. Ook heb ik de style-guide gebruikt om te kijken hoe de stijl van Brainport Eindhoven wordt toegepast binnen verschillende mediaproducten.

Daarna ben ik gaan kijken naar verschillende voorbeelden van nieuwsbrief aanmeld meldingen en formulieren. Het bleek dat de meeste van deze formulieren uniform zijn op de stijl na. De voorbeelden die ik bekeken heb zijn op dit Pinterest bord terug te vinden.

De voorbeelden en de stijl van Brainport Eindhoven heb ik samengebracht ik een prototype. Dit prototype heb ik laten testen door de gebruiker een aantal opdrachten te geven om uit te voeren. De resultaten van deze test heb ik weer verder kunnen verwerken.

De onderbouwing voor mijn keuzes zijn hier terug te vinden.

Reflectie

Door het maken van deze opdracht heb ik kunnen nadenken over mediaproducten zijn opgebouwd en welke (ongeschreven) regels worden toegepast op bepaalde onderdelen zoals het menu van een website bijvoorbeeld. Het is belangrijk om te weten hoe mediaproducten worden opgebouwd omdat hierbij de gebruikerservaring van groot belang is.

Ik heb dit prototype laten testen door verschillende gebruikers waarbij ik vertelde dat ze niet Hierbij kwam uit dat de formuliervelden niet interactief waren en dat niet alle elementen clickable waren. Dit komt doordat de software waarmee ik het prototype gebouwd heb niet over de functies beschikt. Maar ook kreeg ik als feedback mee dat ze niet wisten waar ze moesten klikken om de notificatie weg te klikken en dat de verschillende teksten op knoppen verwarrend was. Op deze feedback heb ik dan ook weer een iteratie gemaakt waarbij deze problemen verholpen worden. Alhoewel het maar een kleine aanpassing is, kan hieronder het resultaat bekeken worden.
